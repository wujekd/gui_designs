When you run the suggested git push --set-upstream origin master command, it not only pushes your changes to GitHub but also establishes a connection between your local master branch and the remote master branch on GitHub. After setting the upstream branch, future pushes and pulls will be simpler as Git will remember the association.

History

This command tells Git to push the changes to the master branch on the remote repository (origin), and it sets up the tracking relationship between your local master branch and the remote master branch on GitHub.

History

Git is informing you that the current branch (master) doesn't have an upstream branch associated with it. An upstream branch is the remote branch on GitHub that corresponds to your local branch.
